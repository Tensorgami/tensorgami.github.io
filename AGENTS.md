# Tensorgami site instructions

This is a static GitHub Pages site. There is no build system.

Primary files:

- `index.html`: homepage/archive
- `about.html`: about page
- `mechanisms.html`: mechanism map
- `verification.html`: verified artifacts index
- `notes/`: note landing pages
- `styles.css`: global styling
- `pdfs/`: mathematical PDFs
- `formalizations/`: Lean sources
- `portfolio.json`, `llms.txt`: machine-readable summaries

Site identity:

Tensorgami is a public mathematical workbench. Its mandate is to make
mathematical mechanisms explicit, algorithmic, concrete, and inspectable.

Style principles:

- Professional, sober, and polished.
- Avoid polemics and manifesto bloat.
- Emphasize mechanisms over subject breadth.
- Prefer direct language: normal forms, reductions, obstruction, rigidity,
  finite certificates, algorithmic extraction, quantitative structure, descent,
  factorization, formal verification.
- Do not introduce JavaScript unless explicitly requested.
- Preserve all existing PDF and Lean links unless explicitly instructed.
- Use accessible semantic HTML.
